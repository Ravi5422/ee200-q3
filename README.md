# 🎵 Sonic Signatures & Zapptain America

**EE200 — Quarter 3: Audio Fingerprinting Project**

A Shazam-clone audio fingerprinting system that identifies songs from short, noisy query clips. The pipeline converts raw audio into compact spectral fingerprints, stores them in a searchable database, and matches unknown queries against the database using histogram-based scoring.

---

## 📁 Directory Structure

```
ee200_q3/
├── fingerprint.py          # Core fingerprinting engine (spectrogram, constellation, hashing)
├── build_database.py       # CLI tool to build the fingerprint database from a song library
├── report_generators.py    # Generates figures and visualizations for the written report
├── app.py                  # Streamlit web app for interactive song identification
├── database.pkl            # Pre-built fingerprint database (generated by build_database.py)
├── requirements.txt        # Python dependencies
├── songs/                  # Directory of full-length reference songs (.wav / .mp3)
├── queries/                # Directory of short query clips to identify
└── figures/                # Output directory for report figures and plots
```

## 🔧 Usage

### 1. Build the Fingerprint Database

Scan every song in the `songs/` directory, extract fingerprints, and serialize them to `database.pkl`:

```bash
python build_database.py --songs_dir ./songs
```

This only needs to be run once (or whenever the song library changes).

### 2. Generate Report Figures

Produce spectrograms, constellation maps, and other visualizations used in the written report:

```bash
python report_generators.py --songs_dir ./songs
```

Figures are saved to the `figures/` directory.

### 3. Run the Streamlit App

Launch the interactive web interface for uploading and identifying query clips:

```bash
streamlit run app.py
```

The app will open in your default browser at `http://localhost:8501`.

## ☁️ Streamlit Cloud Deployment

Deploying to [Streamlit Community Cloud](https://streamlit.io/cloud) is straightforward:

1. Push the entire repository to GitHub, **including `database.pkl`** so the app
   doesn't need to rebuild the database on every cold start.
2. On Streamlit Cloud, create a new app and point it to your repository.
3. Set **`app.py`** as the main file.
4. Streamlit Cloud will automatically detect `requirements.txt` and install the
   listed dependencies.

That's it — no Docker, no server configuration.

## 🧠 Methodology

The fingerprinting pipeline follows four stages:

1. **Spectrogram** — Each audio file is converted into a time–frequency
   representation using the Short-Time Fourier Transform (STFT). This reveals
   how spectral energy evolves over time.

2. **Constellation Map** — Local peaks (high-energy time–frequency points) are
   extracted from the spectrogram. These sparse landmark points are robust to
   noise and volume changes.

3. **Paired Hashing** — Nearby constellation points are grouped into pairs, and
   each pair is hashed into a compact fingerprint token that encodes the two
   frequencies, the time delta between them, and an absolute time anchor.

4. **Histogram Matching** — When a query arrives, its fingerprint hashes are
   looked up in the database. For each candidate song, a histogram of time
   offsets is built; the song whose histogram shows the tallest, sharpest peak
   is returned as the match.

## 📋 Output Format

Prediction CSV files must follow a **strict format**:

```
filename,prediction
query_001.wav,song_a
query_002.wav,song_b
```

- Two columns: `filename` and `prediction`.
- **No index column** — do not include row numbers.
- Each row maps a query clip to its predicted matching song.
